# 추천 모델 챕터 (통계 기반, 딥러닝 미사용)

## 목적
사용자의 궤적 지표·신체 특성·현재 민감도를 입력받아 **조작 모드별 최적 민감도 절대값 + 95% 신뢰구간**을 추정.

## 방침
- **딥러닝 사용하지 않음** — 개인 데이터 중심이라 tabular/통계 기법이 안정적이고 설명 가능.
- **모드별 분리 최적화** — 지상전 / 파쿠르 / 에임글라이드 각각 별도 탐색. "종합 최적값 1개"는 Phase 0 unimodal 실증 결과에 따라 재검토.

## 입력 피처
- 사용자 특성: DPI, 마우스패드 사이즈, 현재 민감도, 현재 cm/360
- 궤적 지표 (세션별, **조작 모드별 분리 집계**):
  - 평균 타겟 도달 시간 (ms)
  - 오버슈트율 (%)
  - 마우스 이동 분포: dx/dy 표준편차, 95퍼센타일 반경
- 조작 모드 태그: `{ground, parkour, aimglide}` — 속도·높이·에임다운 플래그에서 추출
- 주관 평가 (선택): 세션 후 "편했다/불편했다" 1~5점

## Phase 1 — Bisection + A/B/A + SPRT

**알고리즘:**
```
INPUT: 모드 M ∈ {ground, parkour, aimglide}
       초기 구간 [lo, hi] — §5-3 cm/360 매핑으로 설정
       목적함수 f(s) = w1*aim_time + w2*overshoot_rate (낮을수록 좋음)

REPEAT 최대 7회:
    mid = (lo + hi) / 2
    후보 = [lo, mid, hi]  # 3점
    각 후보에 대해 A/B/A 순서 무작위로 짧은 세션(2~3분) 실행

    # SPRT 조기 종료
    두 점 p1, p2 비교:
        누적 관측치로 H0(p1=p2) vs H1(p1≠p2) 우도비 계산
        유의차 확인되면 종료, 아니면 미니 세션 추가

    승자 = argmin f(s)
    구간 축소: 승자가 lo 쪽이면 hi=mid, hi 쪽이면 lo=mid, mid면 [lo, hi] 폭 절반으로

    IF (hi - lo) < 임계치: 종료

OUTPUT: 최종 s* + 95% CI + 탐색 경로 전체
```

**핵심 원칙:**
- **측정 노이즈 대응:** SPRT(Sequential Probability Ratio Test)로 두 점 비교 시 유의차 나올 때까지만 세션 수집. CI 겹치면 "무승부 → 재측정".
- **학습 효과 대응:** A/B/A 3세션 순서 무작위, 두 번의 A 평균과 B 비교.
- **Unimodal 가정:** Phase 0에서 지상전 grid scan으로 실증 후 시작.
- **피로도 제약:** 세션 길이 2~3분으로 단축. 하루 할당량 제한.

**데이터 요구량:** 없음. 개인 데이터만으로 수렴.

**금지:**
- MAB(Thompson Sampling) — 연속 1D unimodal 구조를 낭비함. golden-section이 $O(\log n)$으로 이론적 최적.
- 단일 세션 비교로 승자 결정 — 분산 때문에 flip-flop.

## Phase 2 — XGBoost 회귀 (다중 사용자 일반화, 선택)

**역할:** Phase 1 Bisection의 **초기 구간을 개인별로 좁게 제안**해 탐색 세션 수 절감.

- 입력: 사용자 특성(DPI, cm/360, 손 크기 추정, 기존 선호 민감도)
- 출력: 조작 모드별 초기 구간 중심값 `s0_ground, s0_parkour, s0_aimglide`
- 학습 데이터: 다른 사용자들의 Phase 1 **탐색 경로 전체**(수렴값 + 패배 구간 포함)
- 필요 규모: 사용자 10+ × 완료 탐색 2+ = 세션 수백 개

**왜 XGBoost인가:**
- Tabular 소규모 데이터에서 DL보다 안정적·해석 가능
- Feature importance로 "어떤 특성이 민감도를 결정하는가" 설명 가능

## Phase 3 — 딥러닝 미사용
확장 필요 시 재검토. 현재 로드맵에서 제외.

## 평가 방식
- **동일 사용자 내부 비교:** 여러 민감도로 플레이 → 궤적 지표 차이
- **Ground truth:** "내가 가장 편하게 느낀 민감도" 주관 라벨 + 객관 지표(도달 시간 최소 + 오버슈트율 최소)
- **Phase 2 검증:** XGBoost가 제안한 초기 구간이 실제 Bisection 수렴값을 포함하는 비율(coverage)

## 주의
- **모집단 편향:** 초기 참여자는 워프레임 베테랑 위주. Phase 2 XGBoost는 일반 사용자 데이터가 쌓일 때까지 홀드.
- **개인화 미세조정 우선:** 같은 사용자의 여러 세션으로 개인 튜닝 — Phase 1 Bisection 결과를 그대로 신뢰.
- **Phase 0 실증 전제:** 슬라이더 단조성 + 지상전 unimodal 두 가지가 실측으로 확인되지 않으면 이 알고리즘 전체가 흔들린다.
