# 추천 모델 (ML) 챕터

## 목적
사용자의 궤적 지표·신체 특성·현재 민감도를 입력받아 **최적 민감도 절대값**을 추정.

## 입력 피처
- 사용자 특성: DPI, 마우스패드 사이즈, 현재 민감도, 현재 cm/360
- 궤적 지표 (세션별):
  - 평균 타겟 도달 시간 (ms)
  - 오버슈트율 (%)
  - 마우스 이동 분포: dx/dy 표준편차, 95퍼센타일 반경
- 주관 평가 (선택): 세션 후 "편했다/불편했다" 1~5점

## 모델 선택
- **Phase 1 — 휴리스틱**: 오버슈트율 > 임계치 → 민감도 하향 / 도달 시간 느림 → 민감도 상향
- **Phase 2 — 베이스라인 ML**: XGBoost 회귀. 입력 → 출력: `Δ민감도 = optimal - current`
- **Phase 3 — 딥러닝 (선택)**: 궤적 자체를 인코딩하려면 1D CNN 또는 작은 Transformer

## 평가 방식
- 동일 사용자가 **여러 민감도로 플레이 → 궤적 지표 비교**
- "내가 가장 편하게 느낀 민감도"를 ground truth로 사용
- 주관 라벨만으로는 부족하니 객관 지표(도달 시간 최소 + 오버슈트율 최소)를 보조 목표로

## 데이터 요구량
- 휴리스틱: 필요 없음
- XGBoost: 최소 수백 세션 (사용자 10+ × 세션 20+ 정도)
- DL: 그 이상

## 주의
- **모집단 편향**: 초기 참여자는 워프레임 베테랑 위주일 것. 범용 추천은 데이터가 다양해질 때까지 홀드.
- 개인화 미세조정이 범용 모델보다 우선 — 동일 사용자의 여러 세션으로 개인 튜닝.
